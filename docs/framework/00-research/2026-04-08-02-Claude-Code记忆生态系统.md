│                                                                             │
│  Worker Service (HTTP API: 37777):                                          │

│                                                                             │

│  ├── SQLite Database   → sessions, observations, summaries                  │
│  ├── FTS5 全文搜索     → 快速文本检索                                        │

│                                                                             │

| **Multi-language** | AI理解任何语言的纠正 |

#### 两阶段流程

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  Stage 1        │     │  Stage 2        │     │  Result         │
│  Capture        │     │  Process        │     │                 │
├─────────────────┤     ├─────────────────┤     ├─────────────────┤
│  自动Hooks:     │     │  手动运行:      │     │  同步到:        │
│  • session_start│     │  /reflect       │     │  • CLAUDE.md    │
│  • capturelearn │────→│  → 人工审核     │────→│  • AGENTS.md    │
│  • check_learn  │     │  → 应用/编辑/跳过│     │  • skill files  │
│  • post_commit  │     │                 │     │                 │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

#### 检测方法

1. **Regex Patterns** (实时捕获)
   - 纠正: `"no, use X"` / `"don't use Y"` / `"actually..."`
   - 正反馈: `"Perfect!"` / `"Exactly right"`
   - 显式标记: `"remember:"` (最高置信度)

2. **Semantic AI Validation** (/reflect时)
   - 多语言支持
   - 更高准确性
   - 置信度评分 (0.60-0.95)

### 2.5 claude-memory-compiler (125 ⭐) - 对话知识编译器

#### 核心理念

> 改编自 Karpathy's LLM Knowledge Base，但数据源是你自己的Claude Code对话

#### 架构

```
Conversation → SessionEnd/PreCompact hooks → flush.py
    → daily/YYYY-MM-DD.md → compile.py → knowledge/concepts/, connections/, qa/
        → SessionStart hook → index.md → 下一个会话
```

#### 核心组件

| 组件 | 功能 |
|------|------|
| **flush.py** | 调用Claude Agent SDK提取有价值信息 |
| **compile.py** | 将日志编译为结构化概念文章 |
| **query.py** | 基于索引的问答 (无需RAG) |
| **lint.py** | 7种健康检查 (断链、孤儿、矛盾、过期) |

#### 为什么不用RAG?

> Karpathy的洞察：在个人规模 (50-500篇文章)，LLM阅读结构化index.md胜过向量相似度。RAG在~2000+文章时才需要。
