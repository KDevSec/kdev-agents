# 三大记忆系统深度对比

> 生成日期：2026-04-07
> 目的：深度对比 OMC、ECC、Claudeception 三大记忆系统，为 KDev-Agent 记忆层选择提供决策依据

---

## 一、记忆系统概览

| 系统 | 所属框架 | 核心定位 | 数据规模 |
|------|----------|----------|----------|
| **OMC Memory** | oh-my-claudecode | 多层次记忆 + 状态持久化 | 3层 + 4类状态文件 |
| **ECC Memory** | Everything Claude Code | 持续学习 + 技能生态 | 125 Skills + 28 Agents |
| **Claudeception** | 独立概念 | 知识提取 + 模式复用 | 动态生成技能文件 |

## 二、OMC 记忆系统详解

### 2.1 核心架构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                     OMC 记忆系统架构                                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Layer 1: Notepad (三层记忆)                                                 │
│  ┌───────────────────────────────────────────────────────────────────────┐ │
│  │  priority/  → 优先级记忆 (始终加载，<500字符)                           │ │
│  │  working/   → 工作记忆 (自动清理，7天过期)                              │ │
│  │  manual/    → 手动记忆 (永不过期，用户显式添加)                         │ │
│  └───────────────────────────────────────────────────────────────────────┘ │
│                              ↓                                              │
│  Layer 2: Project Memory (项目级知识)                                        │
│  ┌───────────────────────────────────────────────────────────────────────┐ │
│  │  techStack/    → 技术栈信息                                            │ │
│  │  build/        → 构建/测试命令                                         │ │
│  │  conventions/  → 编码规范                                              │ │
│  │  structure/    → 项目结构                                              │ │
│  │  notes/        → 分类笔记                                              │ │
│  │  directives/   → 用户指令                                              │ │
│  └───────────────────────────────────────────────────────────────────────┘ │
│                              ↓                                              │
│  Layer 3: State Persistence (状态持久化)                                     │
│  ┌───────────────────────────────────────────────────────────────────────┐ │
│  │  .omc/state/                                                          │ │
│  │  ├── session.json   → 会话状态 (phase, iteration, task)               │ │
│  │  ├── tasks.json     → 任务进度 (owner, status, dependencies)          │ │
│  │  ├── quality.json   → 质量指标 (coverage, errors, warnings)           │ │
│  │  └── agents.json    → Agent状态 (active, idle, completed)             │ │
│  └───────────────────────────────────────────────────────────────────────┘ │
│                              ↓                                              │
│  Layer 4: Skill Learning (技能学习)                                          │
│  ┌───────────────────────────────────────────────────────────────────────┐ │
│  │  /learner → 提取可复用模式                                              │ │
│  │  .omc/skills/*.md → 技能文件 (triggers自动匹配)                        │ │
│  │  自动注入 → 匹配关键词时自动加载                                         │ │
│  └───────────────────────────────────────────────────────────────────────┘ │
│                              ↓                                              │
│  Layer 5: Cross-session Handoff (跨会话交接)                                 │
│  ┌───────────────────────────────────────────────────────────────────────┐ │
│  │  会话结束 → 生成 Handoff 文档                                           │ │
│  │  新会话 → 恢复上下文 + 继续执行                                         │ │
│  └───────────────────────────────────────────────────────────────────────┘ │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 核心工具

| 工具 | 功能 | 使用场景 |
|------|------|----------|
| `notepad_read` | 读取记忆内容 | 会话开始时加载上下文 |
| `notepad_write_priority` | 写入优先级记忆 | 项目级重要信息 (<500字符) |
| `notepad_write_working` | 写入工作记忆 | 临时决策、发现的问题 |
| `notepad_write_manual` | 写入手动记忆 | 用户显式要求记住的内容 |
| `project_memory_read` | 读取项目记忆 | 了解项目背景 |
| `project_memory_write` | 写入项目记忆 | 更新项目知识库 |
| `project_memory_add_note` | 添加分类笔记 | 按类别存储信息 |
| `project_memory_add_directive` | 添加用户指令 | 持久化用户偏好 |
| `state_read/write` | 状态管理 | Pipeline状态持久化 |

### 2.3 技能学习机制

```yaml
# .omc/skills/fix-proxy-crash.md
---
name: Fix Proxy Crash
description: aiohttp proxy crashes on ClientDisconnectedError
triggers: ["proxy", "aiohttp", "disconnected"]
source: extracted
---
Wrap handler at server.py:42 in try/except ClientDisconnectedError...
```

**特点：**
- **触发器匹配**：关键词自动加载相关技能
- **双层作用域**：项目级 (`.omc/skills/`) + 用户级 (`~/.omc/skills/`)
- **优先级**：项目级 > 用户级
- **自动学习**：`/learner` 提取模式并生成技能文件

### 2.4 状态持久化

**六类状态追踪：**

| 状态类型 | 文件 | 内容 |
|----------|------|------|
| 会话状态 | session.json | phase, iteration, task_description |
| 任务状态 | tasks.json | owner, status, dependencies |
| 质量状态 | quality.json | coverage, errors, warnings |
| Agent状态 | agents.json | active, idle, completed |
| 记忆状态 | notepad.md | priority, working, manual |
| 项目记忆 | project-memory.json | techStack, build, conventions |

## 四、Claudeception 概念详解

### 4.1 核心概念

Claudeception 是一种"自我反思提取"的概念模式，源自 OMC 文档中的描述：

```
Session 执行 → 发现非显性知识 → 提取模式 → 生成技能文件
                                                    ↓
                              .omc/skills/extracted-skill.md
                                                    ↓
                              后续会话自动注入
```

### 4.2 自动提取流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                     Claudeception 自动提取流程                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Step 1: 执行任务                                                           │
│  ┌───────────────────────────────────────────────────────────────────────┐ │
│  │  Agent 执行复杂任务 → 遇到非显性知识 → 解决问题                         │ │
│  └───────────────────────────────────────────────────────────────────────┘ │
│                              ↓                                              │
│  Step 2: 识别模式                                                           │
│  ┌───────────────────────────────────────────────────────────────────────┐ │
│  │  反思: "这个问题是怎么解决的?"                                          │ │
│  │  提取: 解决步骤 + 关键决策 + 避坑指南                                   │ │
│  └───────────────────────────────────────────────────────────────────────┘ │
│                              ↓                                              │
│  Step 3: 生成技能文件                                                        │
│  ┌───────────────────────────────────────────────────────────────────────┐ │
│  │  name: 解决方案名称                                                     │ │
│  │  description: 问题描述                                                  │ │
│  │  triggers: ["关键词", "触发器"]                                         │ │
│  │  source: extracted                                                     │ │
│  │  ---                                                                   │ │
│  │  解决方案内容...                                                        │ │
│  └───────────────────────────────────────────────────────────────────────┘ │
│                              ↓                                              │
│  Step 4: 持久化存储                                                          │
│  ┌───────────────────────────────────────────────────────────────────────┐ │
│  │  保存到 .omc/skills/ 目录                                               │ │
│  │  后续会话通过 triggers 自动匹配加载                                      │ │
│  └───────────────────────────────────────────────────────────────────────┘ │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4.3 与OMC的关系

Claudeception 是 OMC `/learner` 功能的**概念描述**，实际实现：

| 概念 | 实现 |
|------|------|
| Claudeception | OMC 的 `/learner` skill |
| 自动提取 | 需要用户调用 `/learner` |
| 技能文件 | `.omc/skills/*.md` |
| 自动注入 | triggers 关键词匹配 |

## 六、融合建议

### 6.1 推荐方案：OMC + ECC 融合

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                     KDev 记忆层融合架构                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  来自 OMC:                                                                   │
│  ├── Notepad (priority/working/manual 三层记忆)                             │
│  ├── Project Memory (techStack/build/conventions/structure/notes/directives)│
│  ├── State Persistence (6类状态文件)                                        │
│  ├── Skill Learning (/learner + triggers自动注入)                          │
│  └── Cross-session Handoff (跨会话交接)                                     │
│                                                                             │
│  来自 ECC:                                                                   │
│  ├── 语言规范 (12种语言按需加载)                                            │
│  ├── AgentShield (安全扫描 + CVE检测)                                       │
│  └── 技能库参考 (125 Skills模式，可选引入部分技能)                           │
│                                                                             │
│  KDev独创:                                                                   │
│  ├── questions-log (决策追溯 + 纠正参考)                                    │
│  ├── Delta Spec Archive (增量变更归档)                                      │
│  └── Correction Memory (纠正历史沉淀)                                       │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 6.2 融合理由

1. **OMC提供最完整的记忆架构**
   - 5层记忆体系已成熟
   - 6类状态追踪覆盖全场景
   - Handoff跨会话恢复已验证

2. **ECC补充安全和语言能力**
   - AgentShield安全防护
   - 12种语言规范
   - 技能库模式参考

3. **Claudeception概念已包含在OMC中**
   - OMC的`/learner`就是Claudeception的实现
   - 无需单独引入

### 6.3 实施路线

| 阶段 | 内容 | 来源 | 周期 |
|------|------|------|------|
| **Phase 1** | Notepad + Project Memory | OMC | 3天 |
| **Phase 2** | State Persistence | OMC | 3天 |
| **Phase 3** | Skill Learning + Handoff | OMC | 4天 |
| **Phase 4** | 语言规范 (TypeScript/Python) | ECC | 2天 |
| **Phase 5** | AgentShield集成 | ECC | 3天 |
| **Phase 6** | questions-log + Archive | KDev独创 | 3天 |
| **总计** | | | **18天** |

## 七、决策建议

### 7.1 最终推荐

**以OMC记忆系统为底座，融合ECC语言规范和安全防护**

### 7.2 推荐理由

| 维度 | 评分 | 说明 |
|------|------|------|
| **完整性** | ⭐⭐⭐⭐⭐ | 5层记忆 + 6类状态 |
| **成熟度** | ⭐⭐⭐⭐⭐ | 25k stars，生产验证 |
| **可集成性** | ⭐⭐⭐⭐⭐ | TypeScript实现，易改造 |
| **安全性** | ⭐⭐⭐⭐ | 融合ECC AgentShield |
| **语言支持** | ⭐⭐⭐⭐ | 融合ECC语言规范 |

### 7.3 不单独引入Claudeception的理由

1. **概念已实现**：OMC的`/learner`就是Claudeception的实现
2. **功能覆盖**：triggers机制 + 技能文件格式已满足需求
3. **避免重复**：无需引入两套相似机制
