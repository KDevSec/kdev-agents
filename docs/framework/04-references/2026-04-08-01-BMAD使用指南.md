# BMad 使用指南

## 什么是 BMad？

**BMad (BMAD Method)** 是一套**结构化的AI驱动开发方法论**，将软件开发划分为清晰的阶段，每个阶段有专门的技能（Skill）和代理（Agent）来执行。核心理念是：

> **"先规划，后执行"** — 通过严格的阶段划分和产出物验证，确保开发过程有序可控。

---

## 核心工作流 - BMad Method (BMM)

BMad Method 将开发分为 **4个阶段**：

```
┌─────────────────────────────────────────────────────────────────┐
│  Phase 1: 分析 (Analysis)        Phase 2: 规划 (Planning)       │
│  ┌──────────────┐                ┌──────────────┐               │
│  │ 头脑风暴 BP  │  ──────────→   │ 创建PRD CP ★ │ (必需的第一步) │
│  │ 市场研究 MR  │                │ 验证PRD VP   │               │
│  │ 领域研究 DR  │                │ UX设计 CU    │               │
│  │ 技术研究 TR  │                │ 编辑PRD EP   │               │
│  │ 产品简报 CB  │                └──────────────┘               │
│  └──────────────┘                                                │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  Phase 3: 方案设计 (Solutioning)                                │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐       │
│  │ 创建架构 CA ★│ →  │ 创建史诗故事CE★│ →  │ 实现就绪检查IR★│       │
│  └──────────────┘    └──────────────┘    └──────────────┘       │
│  (★ = 必需步骤)                                                  │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  Phase 4: 实现 (Implementation)                                 │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │                    故事开发循环                            │  │
│  │  Sprint计划 SP★ → 创建故事 CS★ → 开发故事 DS★ → 代码审查 CR │  │
│  │       ↑                                      ↓            │  │
│  │       └←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←│  │
│  │                              有问题则返回修复               │  │
│  │                              通过则下一个故事               │  │
│  │                              Epic完成则回顾 ER             │  │
│  └──────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

## 如何开始使用？

### 场景一：全新项目

**推荐路径：**

| 步骤 | 技能 | 命令 | 说明 |
|------|------|------|------|
| 1 | 产品简报 | `/bmad-product-brief` | 明确产品愿景（可选） |
| 2 | 创建PRD | `/bmad-create-prd` | **必需** - 产品需求文档 |
| 3 | UX设计 | `/bmad-create-ux-design` | 如果涉及UI界面 |
| 4 | 技术架构 | `/bmad-create-architecture` | **必需** - 技术决策文档 |
| 5 | 史诗与故事 | `/bmad-create-epics-and-stories` | **必需** - 拆分开发任务 |
| 6 | 就绪检查 | `/bmad-check-implementation-readiness` | **必需** - 验证准备工作 |
| 7 | Sprint计划 | `/bmad-sprint-planning` | **必需** - 开始开发 |

### 场景二：现有项目（Brownfield）

**推荐路径：**

| 步骤 | 技能 | 命令 | 说明 |
|------|------|------|------|
| 1 | 项目上下文 | `/bmad-generate-project-context` | 扫描代码生成上下文 |
| 2 | 项目文档 | `/bmad-document-project` | 分析现有项目结构 |

### 场景三：快速任务

| 技能 | 命令 | 适用场景 |
|------|------|----------|
| Quick Dev | `/bmad-quick-dev` | 小任务、一次性开发、不需要完整规划 |

## 核心技能详解

### Phase 1 - 分析阶段

| 代码 | 技能名 | 用途 |
|------|--------|------|
| `[BP]` | Brainstorm Project | 多种头脑风暴技术引导 |
| `[MR]` | Market Research | 市场分析、竞争格局 |
| `[DR]` | Domain Research | 行业领域专业知识 |
| `[TR]` | Technical Research | 技术可行性评估 |
| `[CB]` | Create Brief | 产品概念精炼 |

### Phase 2 - 规划阶段

| 代码 | 技能名 | 用途 | 必需? |
|------|--------|------|-------|
| `[CP]` | Create PRD | 产品需求文档 | **★ 必需** |
| `[VP]` | Validate PRD | PRD质量验证 | |
| `[EP]` | Edit PRD | 修改现有PRD | |
| `[CU]` | Create UX | UX/UI设计 | |

### Phase 3 - 方案阶段

| 代码 | 技能名 | 用途 | 必需? |
|------|--------|------|-------|
| `[CA]` | Create Architecture | 技术架构文档 | **★ 必需** |
| `[CE]` | Create Epics & Stories | 开发任务拆分 | **★ 必需** |
| `[IR]` | Implementation Readiness | 就绪检查 | **★ 必需** |

### Phase 4 - 实现阶段

| 代码 | 技能名 | 用途 |
|------|--------|------|
| `[SP]` | Sprint Planning | Sprint计划 (**★ 必需**) |
| `[SS]` | Sprint Status | Sprint状态查看 |
| `[CS]` | Create Story | 创建单个开发故事 |
| `[VS]` | Validate Story | 故事验证 |
| `[DS]` | Dev Story | 开发执行 (**★ 必需**) |
| `[CR]` | Code Review | 代码审查 |
| `[QA]` | QA Test | 自动化测试 |
| `[ER]` | Retrospective | Epic回顾 |

## 产出物存储位置

默认配置下，产出物存储结构：

```
项目根目录/
├── _bmad-output/
│   ├── planning-artifacts/      # 规划产出物
│   │   ├── prd.md               # PRD文档
│   │   ├── architecture.md      # 架构文档
│   │   ├── epics-stories.md     # 史诗和故事
│   │   └── ux-design.md         # UX设计
│   │
│   └── implementation-artifacts/  # 实现产出物
│       ├── sprint-status.yaml      # Sprint状态
│       ├── stories/                # 故事文件
│       └── tests/                  # 测试文件
│
└── docs/                          # 项目知识库
```

## 实际使用示例

### 完整项目开发流程

```bash
# 1. 新建项目目录
mkdir my-project && cd my-project

# 2. 创建PRD（首次使用会自动引导配置）
/bmad-create-prd

# 3. 完成PRD后，进入方案阶段
/bmad-create-architecture         # 设计技术架构
/bmad-create-epics-and-stories    # 拆分开发任务
/bmad-check-implementation-readiness  # 检查就绪

# 4. 开始开发循环
/bmad-sprint-planning             # Sprint计划
/bmad-create-story                # 创建第一个故事
/bmad-dev-story                   # 开发故事
/bmad-code-review                 # 代码审查（自动循环）
```

### 快速开发示例

```bash
# 对于小任务或一次性开发
/bmad-quick-dev
```

## 重要提示

1. **每个技能在新的上下文窗口运行** — BMad强烈建议这样做，避免上下文混乱
2. **必需步骤不可跳过** — 标有 `★` 的步骤是关键门控点
3. **产出物驱动** — 每个阶段都会生成文档，下一阶段依赖这些文档
4. **循环迭代** — Phase 4 的故事开发是循环模式，直到所有故事完成

## 配置说明

BMad 配置文件位于 `_bmad/config.yaml`，主要配置项：

```yaml
# 用户配置
user_name: 用户名
communication_language: 中文          # AI响应语言
document_output_language: 中文        # 文档语言

# 产出物路径
output_folder: "{project-root}/_bmad-output"
planning_artifacts: "{project-root}/_bmad-output/planning-artifacts"
implementation_artifacts: "{project-root}/_bmad-output/implementation-artifacts"
project_knowledge: "{project-root}/docs"
```

## 常见问题

### Q: 如何查看当前进度？
运行 `/bmad-help` 或 `/bmad-sprint-status`（如果在实现阶段）

### Q: PRD写完后想修改怎么办？
运行 `/bmad-edit-prd` 或 `/bmad-validate-prd` 验证后再修改

### Q: 开发过程中发现架构有问题？
运行 `/bmad-correct-course` 进行纠偏导航

### Q: 如何跳过某些阶段？
必需步骤（★）不可跳过。可选步骤如UX设计、头脑风暴等可以跳过。
