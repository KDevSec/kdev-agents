# 方案 + 架构评审标准（cap: design，多触点）

> **硬规（启动即执行）**：评审 agent 启动须先 `recall(scope=/staff/reviewer, subject:review:design)` 召回方案/架构评审历史校准，再据本标准打分（记忆底座 v1.0 §5.2）。
> **共用骨架**：百分制 schema / 双重通过条件 / 🔴🟡⚪ 分级 / 评分表输出格式 / 评审循环，一律遵循 [通用评分模板.md](通用评分模板.md)。

---

## 评审对象（多触点单能力）

本能力被**两个 caller gate** 调用，standards 同源、评对象与维度集随 request 切换：

| 触点 | caller gate | 被评产物 | 用哪套维度 |
|---|---|---|---|
| **事前（方案设计）** | `req-architect:g-design-review` | `design.md`（概要+详细技术方案：架构/接口/数据模型/风险）| 维度集 A |
| **开发前（实施计划）** | `dev-engineer:g-plan-review` | `plan.md`（实施计划，Gate-A/B + task 拆分）| 维度集 B |

> request 的 `target` 决定走哪套维度；两套都满分 100、阈值同为 85。

---

## 评分维度

### 维度集 A — 事前方案（评 design.md，5 维 × 20 = 100）

| 维度 | 满分 | checklist |
|---|---|---|
| **技术可行** | 20 | □ 方案技术上能落地 □ 关键技术点有验证/先例 □ 无致命可行性盲区 |
| **扩展性** | 20 | □ 对可预见的需求演进留扩展点 □ 不过度设计（YAGNI）□ 接口/数据模型可演进 |
| **复杂度** | 20 | □ 方案复杂度与问题匹配 □ 无不必要的机件 □ 可理解可维护 |
| **选型** | 20 | □ 技术选型有依据、合理 □ 与既有栈/项目宪法一致 □ 无引入无谓新依赖 |
| **风险** | 20 | □ 主要风险已识别 □ 有缓解/回退预案 □ 高风险点有标注 |

### 维度集 B — 开发前 plan（评 plan.md，4 维 × 25 = 100）

| 维度 | 满分 | checklist |
|---|---|---|
| **与 design 一致** | 25 | □ plan 忠实承接 design.md，无擅自偏离 □ 接口/数据模型与 design 对齐 □ 无 design 未覆盖的暗改 |
| **Gate-A·B 合理** | 25 | □ Gate-A（输入就绪）判据明确 □ Gate-B（验收）判据可测 □ gate 触点位置合理 |
| **模块边界** | 25 | □ 模块/任务边界清晰、低耦合 □ 单一职责 □ 跨模块接口约定明确 |
| **依赖方向** | 25 | □ 依赖方向健康（无环、不倒置）□ 分层依赖单向 □ task 拆分顺序符合依赖 |

---

## 阈值

- **通过阈值 = 85**（关键产物，方案/架构错代价高）。
- 双重通过条件：`通过 = 总分 ≥ 85 AND 🔴阻断 = 0`。

---

## 问题分级（design 专属判定示例，标准见通用模板 §3）

- 🔴 **阻断**：方案技术不可行 / plan 背离 design / 依赖成环 / 架构红线被破 / 高风险无任何预案。
- 🟡 **重要**：扩展性不足、复杂度偏高、选型欠依据、模块边界模糊。
- ⚪ **轻微**：命名、文档完整度、次要风险未标。

## 评分表

输出 `handoffs/reviewer/<gate>.design.score.md`（`<gate>` = g-design-review 或 g-plan-review），格式照 [通用评分模板.md §4](通用评分模板.md)：

```yaml
cap: design
target: design.md          # 或 plan.md（dev:g-plan-review 触点）
total: 88/100
dimensions:
  - {name: 技术可行, score: 18/20}
  - {name: 扩展性,   score: 17/20, issues: [🟡 ...]}
  - {name: 复杂度,   score: 18/20}
  - {name: 选型,     score: 18/20}
  - {name: 风险,     score: 17/20}
issues:
  - {level: 🟡重要, file: design.md, desc: 扩展点偏少, suggest: ...}
verdict: PASS              # total 88≥85 AND 🔴=0 → PASS
```
