### 1.3 工具效率洞察
**问题发现**: 大型会话(>10MB)普遍存在以下问题：
- Read调用频繁(平均120次/会话)，说明探索式开发模式
- Grep/Glob调用过多(>15次)，目标路径不够精确
- 并行调用率为0%，工具几乎全部串行执行

**根因分析**:
1. 用户指令不够具体，导致AI需要"盲探索"
2. 项目结构未在CLAUDE.md中清晰描述
3. AI未利用并行调用优化效率

**典型案例**:
```
会话: kdev-agent/144b8393 (10.9MB)
- Read: 122次 (每次读取1-2个文件)
- Grep: 38次 (多次重复搜索)
- Glob: 27次 (范围搜索而非精确路径)
- 总耗时: 约2小时
```

## 四、规范建议

### 4.1 用户端规范

**DO (应该做的):**
- ✅ 提供具体文件路径 (`src/auth/login.ts`)
- ✅ 明确功能边界 (`添加JWT验证，不含登录UI`)
- ✅ 说明验收标准 (`需通过 tests/auth.test.ts`)
- ✅ 请求长度 >100字符 (提供足够上下文)
- ✅ 使用项目术语 (`参考现有Logger模式`)

**DON'T (避免做的):**
- ❌ 模糊指令 (`帮我看看这个代码`)
- ❌ 无边界 (`优化一下`)
- ❌ 无验收标准 (`改好就行`)
- ❌ 过于简短 (`修复bug`)

**指令模板:**
```
在 [具体文件] 中 [具体动作]，
参考 [现有模式/文件]，
验收标准: [测试文件/运行命令]
```

### 4.2 Claude端规范

**工具调用原则:**
| 原则 | 具体约束 |
|------|----------|
| Glob限制 | 单任务≤5次，优先精确路径 `src/**/*.ts` |
| Grep限制 | 单任务≤10次，结合文件类型过滤 |
| Read合并 | 同类文件一次读取多个 |
| 并行优先 | Read + Glob 同时执行 |

**Skill强制流程:**
```
用户请求 → 检查skill列表 → 匹配skill → 执行skill流程
- 实现任务 → brainstorming skill
- Bug修复 → debugging skill
- 测试 → test-driven-development skill
```

**错误处理原则:**
- 工具超时 >10秒 → 评估是否需要后台执行
- 错误发生 → 先分析根因，再行动
- 用户打断 → 询问是否需要调整方向

### 4.3 项目配置规范 (CLAUDE.md)

```markdown
# 项目规范

## 项目结构概览
src/
  auth/      # 认证模块
  api/       # API层
  utils/     # 工具函数
tests/       # 测试文件

## 工具使用约束
- Glob: 使用精确路径，避免 `**/*` 大范围搜索
- Read: 优先读取关键文件，一次读取≤3个
- Grep: 结合 --type 过滤

## Skill触发规则
- 新功能 → 先调用 brainstorming skill
- Bug修复 → 先调用 debugging skill
- 代码变更 → 验证后调用 kdev-commit

## 验收标准模板
- 单元测试: `npm test`
- 类型检查: `npm run typecheck`
- 构建: `npm run build`
```
