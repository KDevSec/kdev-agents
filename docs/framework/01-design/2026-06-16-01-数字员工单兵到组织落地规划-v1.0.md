## 0. 一句话

数字员工要"像真人一样胜任并被经营"，得先有**会干活的手**、**会配合的同事**、**记得住的脑**、**查得到的知识**，最后被组织像管真人一样**选育留、算价值**——这就是从**一个能干活的数字员工**，长成**一支可被经营的数字员工组织**的落地路径：**5 步、归为 3 大阶段**。前 2 步已真实跑通，当前主战场在第 3-4 步（记忆系统 + 知识库），终点是第 5 步的组织化治理（成熟终态）。

---

## 1. 现状与起点（我们已建成什么）

我们已经设计并建成一套**数字员工技术底座**，不是 PPT 构想，是已端到端跑通真实任务的系统。

**两个底座**
- **编排底座 `kdev-core`** —— 让员工"按流程跑起来"：状态记录 + 节点流转 + 结构化评审 gate，铁律是"只记账 + 流转，不替员工做判断"。
- **记忆底座 `kdev-memory`** —— 让员工"记得住、越用越强"：叙事 / 步骤 / 决策 / 踩坑 / 自评他评，多 scope（shared 共享 / staff 员工私有），Git 托管跨会话续航。

**已建成的角色（6 角色团队）**
- ✅ 已落地为实体员工：**需求架构师**、**开发工程师**、**测试工程师**、**评审专家**（均为可直派的 agent，集中在 `kdev-team` 插件）。
- 主控员 = 主会话总指挥；监督员（CQO 元监督）= 成熟终态阶段（见第 5 步）。

**起点的"成色"（诚实校准）**
- **会干活** ✅ —— 单员工端到端跑通真实改造任务（自主度 L2 协同）。
- **会协作** ✅ —— 多员工分工 + 跨员工交接（handoff）+ 评审专家 + 跨员工发函硬规，已是一个会配合、能互评的小班组。
- **记忆初级** ⚠️ —— 目前靠 md / wiki 形式的蒸馏经验 + scope 分离 + 智能召回，**尚未 OS 化**（结构化检索、组织级共享记忆体仍是升级方向）。
- **无知识库** ❌ —— 还没有领域知识库（RAG / wiki 检索增强）。
- **无组织治理** ❌ —— 还没有"选用育留备退"生命周期管理与价值度量。

> 一句话起点定位：**会干活、会协作的初级数字员工团队，脑子（记忆）还初级、肚子里（知识）还空、上面（组织治理）还没建。**

## 2. 目标愿景（集团数字员工方法论）

终点对齐集团数字员工方法论——一个**类真人的数字员工**，且能被组织**成建制地经营**。

**"我是谁 - 从哪来 - 到哪去" = 单体生命周期三视角**
- **我是谁（角色管理）**：用经营力 / 业务力 / 专业力 + 9 维度（价值贡献度 / 场景覆盖度 / 流程渗透度 / 能力成熟度 / 系统体验度 / 数据完备度 / 模型适配度 / 智算应用度 / 安全可信度）刻画一个数字员工"成熟到什么程度"。
- **从哪来（周期管理）**：用 HR + IR 双线把数字员工纳入**选 - 用 - 育 - 留 - 备 - 退**生命周期，"百人百智"规模化复制。
- **到哪去（运用实例）**：单个数字员工（如"软件代码助理小智"）在真实业务里持续产出、量化价值。

**"新环境 5 保障"** = AI 架构 / 流程 / 能力 / 组织 / 投入保障，对应安全可信度等底线。

> 终点定位：**从"一个小智"到"一支可被组织选育留、能算清价值贡献的数字员工队伍"。**

## 3. 路线为什么是这条：两轴模型

把"单人 → 组织"硬掰成一条线会拧巴——因为我们**起点就已经是多员工团队**。真正干净的拆法是**两根轴**，落地路线沿对角线往右上爬：

<table>
<thead>
<tr><th>维度</th><th>纵轴 ＝ 个体成熟度（像不像真人）</th><th>横轴 ＝ 组织化程度（成不成建制）</th></tr>
</thead>
<tbody>
<tr><td rowspan="4"><b>内涵</b></td><td>会干活（编排）</td><td>单员工</td></tr>
<tr><td>会记事（记忆）</td><td>多员工分工协作</td></tr>
<tr><td>会查知识（知识库）</td><td>员工集群 ＋ 生命周期治理（选用育留备退）</td></tr>
<tr><td>会自省（自演进）</td><td>经营度量</td></tr>
<tr><td rowspan="3"><b>起点（现状）</b></td><td>会干活 ✅</td><td>多员工分工 ✅</td></tr>
<tr><td>记忆初级 ⚠️</td><td>评审 ＋ handoff ✅</td></tr>
<tr><td>无知识库 ❌</td><td>无生命周期 / 无价值度量 ❌</td></tr>
<tr><td rowspan="3"><b>终点（目标）</b></td><td>memOS</td><td>选用育留备退</td></tr>
<tr><td>RAG 知识库</td><td>三力价值度量</td></tr>
<tr><td>自省进化 ＝ 类真人</td><td>安全可信 ＝ 成建制组织</td></tr>
</tbody>
</table>

**到目标态真正缺的三块**（汇报里的"差距清单"，见第 5 节）：① 记忆 OS 化 ② 知识库 ③ 组织治理与价值度量。

## 4. 落地路线：5 步 ⊂ 3 大阶段（主体）

对外讲 **3 个大阶段**（叙事干净），内容主干切 **5 步**（能挂里程碑 + 显性化"已跑通"的执行力）。

```
阶段一：单员工成形（起点·已达成）          阶段二：补全"类真人"（当前主战场）        阶段三：成建制组织（终态）
  ┌──────────────┐                      ┌──────────────────────────┐         ┌──────────────────────┐
  │ 1 会干活      │                      │ 3 会记事（记忆系统 memOS）  │         │ 5 会经营 · 会自省      │
  │ 2 会协作      │  ──────────────────► │ 4 会查知识（知识库 RAG）    │ ──────► │   组织化治理 + 价值度量 │
  └──────────────┘                      └──────────────────────────┘         └──────────────────────┘
   ✅ 已达成 M1/M2/M3                       🔨 进行中 / ⏳ 规划                     ⏳ 终态 L4
```

### 阶段一 · 单员工成形（起点，已达成）

| 步 | 经营语言标题 | 能力内涵 | 对应方法论概念 | kdev 技术支撑 | 当前状态 / 凭据 |
|---|---|---|---|---|---|
| **1** | **会干活** —— 单个数字员工端到端胜任一个岗位 | 专业力雏形 + 单场景业务力跑通；自主度 L2 协同 | 专业力（能力成熟度起步）| `kdev-core` 编排底座（状态 R1 / 节点机 R2 / 三类 gate R3）驱动 13 节点流程跑通真实改造任务 | ✅ **M1**：集群史上首次端到端实测（底座真在驱动，非 prompt 演戏：全程持久续接 + 评审 gate FAIL→修→重评闭环 + 真产出可运行代码）|
| **2** | **会协作** —— 多能力分工，像个会配合、能互评的班组 | 多员工分工 + 上下游交接 + 互评 = 组织雏形 | 多员工团队 + 员工间直接协作 | `kdev-team` 多员工（需求架构师 / 开发 / 测试 / 评审）+ 跨员工 handoff + 评审专家发函 + 发函硬规 | ✅ **M2**（跨员工 handoff + scope 分离）+ **M3**（评审专家上线 + 跨员工协作硬规 = L3 起步）|

> 阶段一的汇报价值：**"已经跑通"——不是规划，是实测。** 这是整份汇报最硬的执行力背书。

### 阶段二 · 补全"类真人"（中间，当前主战场）

| 步 | 经营语言标题 | 能力内涵 | 对应方法论概念 | kdev 技术支撑 | 当前状态 |
|---|---|---|---|---|---|
| **3** | **会记事** —— 组织级记忆系统 | 从"个人随手记"升级到"组织记得住、可共享、可召回"；提升数据完备度 | 数据完备度 | `kdev-memory` 从 md/wiki 蒸馏 + scope 分离 + 智能召回，**升级为 memOS**：结构化存储 + 检索 + 跨员工组织级共享记忆体 | 🔨 **进行中**（已有 scope/召回/Git 托管基础；OS 化结构检索是升级方向）|
| **4** | **会查知识** —— 领域知识库 | 把领域知识沉淀成可检索资产，检索增强让员工"查得到、答得准"；提升模型适配度 | 知识库 / 模型适配度 | **新增** RAG + wiki 知识库：领域文档沉淀 + 向量检索 + 检索增强生成 | ⏳ **规划**（当前缺口，本期重点立项）|

> 阶段二的汇报价值：**这是"为什么还不是终态"的清晰回答**——记忆要 OS 化、知识库要从零建起，正是让单个数字员工"从会干活到像真人"的关键补全。

### 阶段三 · 成建制组织（终点，成熟终态）

| 步 | 经营语言标题 | 能力内涵 | 对应方法论概念 | kdev 技术支撑 | 当前状态 |
|---|---|---|---|---|---|
| **5** | **会经营 · 会自省** —— 数字员工组织化治理 | 选用育留备退全生命周期(HR/IR) + 经营力·业务力·专业力价值度量 + 安全可信 5 保障 + 自演进；百人百智规模复制 | 新物种/新环境 + 周期管理 + 三力（经营/业务/专业）| 监督员（CQO 元监督）+ 自演进机制 + 跨 IDE；价值度量看板（在观测层 `kdev-hud` 基础上扩展）+ 安全可信保障 | ⏳ **终态 L4**（CQO / 自演进 / 跨 IDE 当前 defer，作为愿景终态）|

> 阶段三的汇报价值：**从"一个小智"到"一支可被经营的数字员工队伍"**——组织能像选育留真人一样管理数字员工、并算清每个员工的价值贡献度与 ROI。

## 5. 关键差距与当前主战场

到目标态，技术与方法论上各还差三块，**当前主战场聚焦前两块**：

| 差距 | 现状 | 目标 | 落在第几步 | 优先级 |
|---|---|---|---|---|
| **记忆 OS 化** | md/wiki 蒸馏 + scope 分离 + 召回 | 组织级 memOS：结构化 + 检索 + 跨员工共享记忆体 | 第 3 步 | 🔴 当前主战场 |
| **知识库** | 无 | RAG + wiki：领域知识沉淀 + 检索增强 | 第 4 步 | 🔴 当前主战场 |
| **组织治理 + 价值度量** | 无 | 选用育留备退生命周期 + 三力度量 + 安全可信 | 第 5 步 | 🟡 终态阶段 |

## 6. 里程碑与节奏

**已达成（执行力背书）**

| 里程碑 | 内容 | 状态 |
|---|---|---|
| **M0** | 编排底座核心三件（状态 / 节点机 / 三类 gate）+ Git 托管激活 | ✅ |
| **M1** ★ | 开发工程师端到端跑通真实改造任务 —— 集群史上首次实测（L2 协同）| ✅ |
| **M2** | 第二员工（需求架构师）并发 + 记忆 scope 分离 + 跨员工 handoff | ✅ |
| **M3** | 评审专家上线 + 测试工程师上线 + 跨员工协作硬规 = **L3 起步** | ✅ |

**后续排序（建议）**

```
当前 ──► 第3步 记忆 memOS 化 ──► 第4步 知识库 RAG/wiki ──► 第5步 组织治理 + 价值度量（终态 L4）
         （已有基础，OS 升级）    （从零建，本期重点）       （CQO/自演进/跨IDE，愿景终态）
```

> 节奏原则（承接技术 roadmap 的 walking-skeleton 思路）：**按真实需求逐级长，不一次性把全部基建压在前面**——先把当前价值密度最高、缺口最痛的记忆 OS + 知识库补齐，再爬向组织治理终态。

## 7. 保障（新环境 5 保障）

| 保障 | 落地抓手 |
|---|---|
| **AI 架构保障** | 编排 / 记忆 / 知识 / 观测四底座分工清晰，可独立演进、headless 可装 |
| **AI 流程保障** | 结构化评审 gate + 跨员工发函硬规 + 有界回流（评审给建议不拦截、超限升人工不降级接受）|
| **AI 能力保障** | 自主度阶梯 L1→L4 逐级验证，每步有可验收里程碑 |
| **AI 组织保障** | 选用育留备退生命周期（第 5 步）+ 监督员 CQO 元监督 |
| **AI 投入保障** | 价值贡献度 / ROI 度量看板（观测层扩展），让投入产出可量化 |
