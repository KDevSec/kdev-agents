## 目标
从 Claude Code 会话 JSONL 文件中提取开发过程经验，分析成功模式和失败教训，形成可复用的规范文档。

## 数据位置
```

```

按项目目录组织，每个会话有：
- 主会话文件: `{session-uuid}.jsonl`
- 子代理会话: `{session-uuid}/subagents/agent-{id}.jsonl`

## 数据结构说明
每行一个 JSON 对象，主要类型：

| type | 说明 | 关键字段 |
|------|------|----------|
| `user` | 用户输入 | `message.content`, `timestamp`, `cwd` |
| `assistant` | AI响应 | `message.content[]` (含 thinking) |
| `attachment` | 钩子/附件 | `attachment.content`, `hookName` |
| `tool_result` | 工具执行结果 | `toolUseResult.durationMs`, `filenames` |

## 分析维度

### 1. 时间范围筛选
- 目标: 2026年3月至当前 (约2个月)
- 时间戳字段: `timestamp` (ISO 8601格式)
- 筛选条件: `timestamp >= "2026-03-01"`

### 2. 项目范围
- D:\Works 下所有项目
- 关注: kdev-agents, KDevSec, token-statistics, skills-dev 等

### 3. 分析维度（必需输出）

#### A. 成功模式 (Good Practices)
提取并归类：
- **高效协作模式**: 用户指令清晰度、上下文补充方式
- **工具使用最佳实践**: 高效的 tool call 模式、并行调用策略
- **代码质量实践**: 测试先行、渐进重构、安全编码

- **理解偏差**: AI误解用户意图的场景

- **流程中断**: 任务未完成、中途放弃的原因

#### C. Token 效率分析

- 低消耗但高产出的模式

#### D. 会话质量指标

- 错误恢复成功率

### A.1 会话统计脚本
```python
import json
from pathlib import Path
from datetime import datetime
from collections import defaultdict

sessions_dir = Path("C:/Users/liyong/.claude/projects")

stats = defaultdict(lambda: {
    "count": 0,
    "total_lines": 0,
    "user_msgs": 0,
    "assistant_msgs": 0,
    "tool_calls": 0,
    "first_date": None,
    "last_date": None
})

for project_dir in sessions_dir.iterdir():
    if "Works" not in project_dir.name:
        continue
    for jsonl_file in project_dir.glob("*.jsonl"):
        if "subagents" in str(jsonl_file):
            continue
        stats[project_dir.name]["count"] += 1
        with open(jsonl_file, 'r', encoding='utf-8') as f:
            for line in f:
                try:
                    record = json.loads(line)
                    stats[project_dir.name]["total_lines"] += 1
                    ts = record.get("timestamp", "")
                    if ts:
                        if stats[project_dir.name]["first_date"] is None:
                            stats[project_dir.name]["first_date"] = ts[:10]
                        stats[project_dir.name]["last_date"] = ts[:10]
                    if record.get("type") == "user":
                        stats[project_dir.name]["user_msgs"] += 1
                    elif record.get("type") == "assistant":
                        stats[project_dir.name]["assistant_msgs"] += 1
                except:
                    pass

print(f"项目数: {len(stats)}")
for proj, data in sorted(stats.items()):
    print(f"{proj}: {data['count']}会话, {data['user_msgs']}用户消息, {data['first_date']}~{data['last_date']}")
```

### A.2 关键事件提取
```python
def extract_key_events(jsonl_path):
    """提取会话关键事件"""
    events = []
    with open(jsonl_path, 'r', encoding='utf-8') as f:
        for line_num, line in enumerate(f, 1):
            try:
                record = json.loads(line)
                event = None

                if record.get("type") == "user":
                    content = record.get("message", {}).get("content", "")
                    if isinstance(content, list):
                        texts = [c.get("text", "") for c in content if c.get("type") == "text"]
                        content = " ".join(texts)
                    # 过滤掉元消息
                    if not any(x in content for x in ["<local-command", "<ide_opened", "tool_result"]):
                        event = {"type": "user_request", "content": content[:200]}

                elif record.get("type") == "assistant":
                    msg = record.get("message", {})
                    thinking = ""
                    for c in msg.get("content", []):
                        if c.get("type") == "thinking":
                            thinking = c.get("thinking", "")[:100]
                    if thinking:
                        event = {"type": "ai_thinking", "summary": thinking}

                # 工具调用统计
                if "toolUseResult" in record:
                    duration = record["toolUseResult"].get("durationMs", 0)
                    event = {"type": "tool_result", "duration": duration}

                if event:
                    event["line"] = line_num
                    event["timestamp"] = record.get("timestamp", "")
                    events.append(event)
            except:
                pass
    return events
```

## 附录 B: 分类标准

### B.1 用户指令质量评分
| 分数 | 特征 | 示例 |
|------|------|------|
| 5分 | 目标明确+上下文完整+边界清晰 | "在 src/auth.ts 添加 JWT 验证，使用现有 jwtHelper，需通过测试" |
| 3分 | 目标明确但缺上下文 | "添加登录功能" |
| 1分 | 模糊或歧义 | "帮我看看这个代码" |

### B.2 Claude 响应质量评分
| 分数 | 特征 |
|------|------|
| 5分 | 正确理解+高效执行+完整验证 |
| 3分 | 正确理解但执行效率低 |
| 1分 | 理解偏差或未完成任务 |

### B.3 工具使用效率分类
| 类型 | 定义 | 影响 |
|------|------|------|
| 高效 | ≤3次调用完成任务 | 低token消耗 |
| 正常 | 4-10次调用 | 中等消耗 |
| 低效 | >10次调用或重复探索 | 高消耗 |
