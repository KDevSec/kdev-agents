02 · 编排底座 kdev-core
底座靠三样东西让员工 跑得起来、卡得住、续得上
铁律：只记账 + 流转，不做判断本身——评审 / 跑测试 / 选路线由员工或人来做，结果喂回底座。
① 记两本机器账
控制态 + 事实流水
控制态「我在哪一步」+ 事实流水「发生了啥」，全程原子化落盘（记忆层从流水提炼，见第 3 页）。

最大好处：断点续跑——断电 / 被打断，再起来从上次那步接着干。
② 流程节点引擎
每条业务自己声明节点表
业务自己声明「有哪些步骤、什么顺序」（不写死段数）。底座只提供推进引擎：把关过 → 进下一步；不过 → 退回重做（有上限，不死循环）。
③ 三类把关关卡
核心难点 = 最大卖点
所有把关点都归成三种类型，一套逻辑处理。这是「一套底座跑任意业务」的根：换业务只换节点表 + 关卡配置。
关卡类型干什么谁来判判不过怎么办
评审检查产出质量（代码 / 方案 / 安全…）默认 AI，可改成人或人机双审退回改 → 重审（最多 3 次）；还不过 → 升级交人工
岔路决策在分叉口选一条路（复杂 / 简单走法）默认 AI，岔路口可交给人选哪条走哪条（不重试）
验收跑端到端测试、验真功能自动跑测试，可叠加人复核没过退回修复节点重跑
把关结果以结构化数据记录（不靠猜文本），能程序化查「这关重试几轮、谁判的」；卡住超上限一定升级交人工，绝不偷偷降格放行。
节点表 ＋ 关卡 ＝ 一条流程的 SOP，可自由编排： L0 作者定默认→ L1 用户改配置（开关关卡 · 谁来判 AI↔人 · 重试数 · 跳过 / 重排节点）→ L2 员工自优化（自演进 · 未来）

03 · 记忆底座 kdev-memory
记忆 = 从机器到人的 「提炼 + 加料」，不重复造账
三层数据从机器到人：控制态 / 事实流水是底座的机器账（第 2 页）；记忆只管最上面的叙事层——从流水蒸馏，再加机器给不了的那一维。

发生了啥（events.jsonl）

＋加料

叙事 / 蒸馏 = 记忆
人 · kdev-memory
步骤 · 决策 · 踩坑 + 自评 / 用户评分
一条原则：记忆只加「机器给不了的那一维」（评价 / 经验 / 叙事）；凡是镜像机器原始态的，让位给底座——不造第二本账。
全团队的记忆底座，不是主控员私有
shared（决策 / 踩坑 / 主控员续航）+ staff/<员工>（各员工私有 rollup + 评审经验 → 喂飞轮）。多消费者：主控员续航 · 评审专家越评越准 · 各员工记自己的账。整套 Git 托管自动同步。
两个模式 · 一条派生链
单装＝给人做跨会话续航（含进度，它是唯一来源）；装了 kdev-team＝进度让位 HUD / 底座，记忆收窄成「判断 + 续航」层。Step 的「执行事实」从流水蒸馏，「自评 / 评分」才是记忆独有的加料。

+未来：也反哺「流程本身」（自演进）
