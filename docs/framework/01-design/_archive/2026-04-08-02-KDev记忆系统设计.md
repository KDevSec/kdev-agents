## 一、设计原则

### 1.0 Compound Engineering 理念

> **核心理念**：Each unit of engineering work should make subsequent units easier—not harder.
> (每一单位工程工作应该让后续工作更容易，而不是更难)

```
传统工程 vs 复利工程
┌─────────────────────────────────────────────────────────────────────────────┐
│  传统工程: 每个功能 → 下一个更难                                             │
│  ├── 代码越多 = 边缘情况越多                                                │
│  ├── 更多依赖和耦合                                                        │
│  └── 积累技术债务                                                          │
│                                                                             │
│  复利工程: 每个功能 → 下一个更容易                                           │
│  ├── bug/测试失败/洞察 → 文档化                                            │
│  ├── 知识资产积累                                                          │
│  └── agent 可以复用先验知识                                                 │
└─────────────────────────────────────────────────────────────────────────────┘
```

**Compound Engineering 四步循环**:
```
Plan (80%) → Work (20%) → Review → Compound → Repeat
                                     ↑
                              知识沉淀 (记忆系统的核心)
```

**KDev 记忆层 = Compound 步骤的实现**：
- 将 Plan/Work/Review 中学到的内容记录下来
- 让下一个会话/agent 可以复用
- 实现知识复利效应

### 1.1 核心定位

```
记忆层 = speckit 的"运行时经验层" = Compound Engineering 的知识沉淀层

speckit 规划: "我们要做什么" (主动规划) → Plan 阶段
记忆层 记录: "我们学到了什么" (被动积累) → Compound 阶段
```

### 1.2 与 speckit 的边界

| 功能 | 归属 | 说明 |
|------|------|------|
| 架构设计 | speckit architecture | 项目规划 |
| 需求规格 | speckit specs | 需求定义 |
| 任务计划 | speckit plans | 任务分解 |
| 决策追溯 | **记忆层** questions-log | 运行时决策 |
| 纠正历史 | **记忆层** corrections | 纠正记录 |
| 踩坑记录 | **记忆层** gotchas | 运行时发现 |
| 工作总结 | **记忆层** daily-logs | 每日回顾 |
| Skill提取 | **记忆层** skills | 模式复用 |

### 1.3 不做什么

- ❌ 全量对话存储 (噪音多，价值低)
- ❌ 向量库 (个人规模不需要)
- ❌ 重复 speckit 已有功能
- ❌ 复杂的记忆检索系统

# JWT 认证快速搭建

## 前置条件
- Node.js 项目
- 已安装 TypeScript

## 步骤

### 1. 安装依赖
```bash
npm install jose
npm install -D @types/node
```

### 2. 创建 JWT 工具
创建 `src/auth/jwt.ts`:
```typescript
import { SignJWT, jwtVerify } from 'jose'

const secret = new TextEncoder().encode(process.env.JWT_SECRET)

export async function signToken(payload: object) {
  return new SignJWT(payload)
    .setProtectedHeader({ alg: 'HS256' })
    .setExpirationTime('1h')
    .sign(secret)
}

export async function verifyToken(token: string) {
  const { payload } = await jwtVerify(token, secret)
  return payload
}
```

### 3. 创建中间件
创建 `src/auth/middleware.ts`:
```typescript
import { verifyToken } from './jwt'

export async function authMiddleware(request: Request) {
  const token = request.headers.get('Authorization')?.split(' ')[1]
  if (!token) throw new Error('Unauthorized')
  return verifyToken(token)
}
```

### 4. 编写测试
创建 `tests/auth.test.ts`:
```typescript
import { signToken, verifyToken } from '../src/auth/jwt'

test('sign and verify token', async () => {
  const token = await signToken({ userId: '123' })
  const payload = await verifyToken(token)
  expect(payload.userId).toBe('123')
})
```

## 验证

```

## 注意事项
- 确保 JWT_SECRET 环境变量已设置
- expiresIn 使用字符串格式如 '1h' 避免歧义
```

### 5.3 SessionStart Hook 实现

**目的**: 会话启动时自动加载记忆注入到上下文

```javascript
// scripts/kdev-session-start.mjs

import { existsSync, readFileSync } from 'fs';
import { join, dirname } from 'path';
import { homedir } from 'os';
import { fileURLToPath, pathToFileURL } from 'url';

const __filename = fileURLToPath(import.meta.url);
const __dirname = dirname(__filename);

// 导入 stdin 读取器 (防止 hang，参考 OMC issue #240)
let readStdin;
try {
  const mod = await import(pathToFileURL(join(__dirname, 'lib', 'stdin.mjs')).href);
  readStdin = mod.readStdin;
} catch {
  readStdin = (timeoutMs = 5000) => new Promise((resolve) => {
    const chunks = [];
    let settled = false;
    const timeout = setTimeout(() => {
      if (!settled) { settled = true; process.stdin.destroy(); resolve(Buffer.concat(chunks).toString('utf-8')); }
    }, timeoutMs);
    process.stdin.on('data', (chunk) => chunks.push(chunk));
    process.stdin.on('end', () => { if (!settled) { settled = true; clearTimeout(timeout); resolve(Buffer.concat(chunks).toString('utf-8')); } });
    process.stdin.on('error', () => { if (!settled) { settled = true; clearTimeout(timeout); resolve(''); } });
  });
}

// 安全读取 JSON 文件
function readJsonFile(path) {
  try {
    if (!existsSync(path)) return null;
    return JSON.parse(readFileSync(path, 'utf-8'));
  } catch { return null; }
}

// 安全读取 Markdown 文件 (前 N 行)
function readMdFile(path, maxLines = 50) {
  try {
    if (!existsSync(path)) return null;
    const content = readFileSync(path, 'utf-8');
    const lines = content.split('\n').slice(0, maxLines);
    return lines.join('\n');
  } catch { return null; }
}

// 收集记忆上下文
async function collectMemoryContext(workingDirectory, sessionId) {
  const home = homedir();
  const messages = [];

  // 1. 加载全局用户偏好 (~/.kdev/user-preferences.md)
  const globalPrefsPath = join(home, '.kdev', 'user-preferences.md');
  const globalPrefs = readMdFile(globalPrefsPath, 30);
  if (globalPrefs) {
    messages.push(`<kdev-memory-global>
# 用户偏好 (全局)
${globalPrefs}
</kdev-memory-global>`);
  }

  // 2. 加载全局纠正历史 (~/.kdev/global-corrections.md)
  const globalCorrectionsPath = join(home, '.kdev', 'global-corrections.md');
  const globalCorrections = readMdFile(globalCorrectionsPath, 20);
  if (globalCorrections) {
    messages.push(`<kdev-memory-corrections>
# 纠正历史 (全局)
${globalCorrections}
</kdev-memory-corrections>`);
  }

  // 3. 加载项目记忆 (.kdev/)
  const kdevDir = join(workingDirectory, '.kdev');
  if (existsSync(kdevDir)) {
    // questions-log (未解决的问题)
    const questionsPath = join(kdevDir, 'questions-log.md');
    const questions = readMdFile(questionsPath, 40);
    if (questions) {
      // 只注入 "待解决问题" 部分
      const pendingMatch = questions.match(/## 待解决问题[\s\S]*?(?=## 已解决|## 纠正|$)/);
      if (pendingMatch && pendingMatch[0].trim().length > 20) {
        messages.push(`<kdev-memory-questions>
# 待解决问题
${pendingMatch[0]}
</kdev-memory-questions>`);
      }
    }

    // gotchas (踩坑记录)
    const gotchasPath = join(kdevDir, 'gotchas.md');
    const gotchas = readMdFile(gotchasPath, 30);
    if (gotchas) {
      messages.push(`<kdev-memory-gotchas>
# 踩坑记录
${gotchas}
</kdev-memory-gotchas>`);
    }

    // session-state (会话状态恢复)

    if (sessionState && sessionState.sessionId === sessionId) {

    }
  }

  // 4. 检查是否有未完成的每日日志
  const today = new Date().toISOString().split('T')[0];
  const dailyLogPath = join(kdevDir, 'daily-logs', `${today}.md`);

</kdev-daily-pending>`);
    }
  }

// 原子写入 JSON (防止并发写入损坏)
function atomicWriteJson(path, data) {
  const tmpPath = path + '.tmp.' + process.pid;
  writeFileSync(tmpPath, JSON.stringify(data, null, 2));
  try {
    // rename 是原子操作
    const fs = require('fs');
    fs.renameSync(tmpPath, path);
  } catch {
    // Windows 可能失败，直接覆盖
    writeFileSync(path, JSON.stringify(data, null, 2));
    try { require('fs').unlinkSync(tmpPath); } catch {}
  }
}

// 收集会话统计
async function collectSessionStats(workingDirectory) {
  const stats = {
    commits: 0,
    filesChanged: 0,
    additions: 0,
    deletions: 0,
    artifacts: []
  };

  // Git 统计
  try {
    const { execSync } = require('child_process');
    const gitLog = execSync('git log --oneline --since="today" 2>/dev/null', { cwd: workingDirectory, encoding: 'utf-8' });
    stats.commits = gitLog.trim().split('\n').filter(l => l).length;

    const gitDiff = execSync('git diff --shortstat HEAD~' + stats.commits + ' 2>/dev/null', { cwd: workingDirectory, encoding: 'utf-8' });
    const match = gitDiff.match(/(\d+) files changed, (\d+) insertions, (\d+) deletions/);
    if (match) {
      stats.filesChanged = parseInt(match[1]);
      stats.additions = parseInt(match[2]);
      stats.deletions = parseInt(match[3]);
    }
  } catch {}

  // 收集产物 (specs, plans, skills)
  const kdevDir = join(workingDirectory, '.kdev');
  if (existsSync(kdevDir)) {
    const skillsDir = join(kdevDir, 'skills');
    if (existsSync(skillsDir)) {
      const skills = readdirSync(skillsDir).filter(f => f.endsWith('.md'));
      stats.artifacts.push(...skills.map(s => `skill: ${s}`));
    }
  }

  const specsDir = join(workingDirectory, 'specs');
  if (existsSync(specsDir)) {
    const specs = readdirSync(specsDir).filter(f => f.endsWith('.md'));
    stats.artifacts.push(...specs.map(s => `spec: ${s}`));
  }

  return stats;
}

### 5.6 纠正检测 Hook (PostToolUse)

**目的**: 检测用户纠正并自动记录到 questions-log

```javascript
// scripts/kdev-correction-detector.mjs

import { existsSync, readFileSync, appendFileSync, mkdirSync } from 'fs';
import { join } from 'path';

import { readStdin } from './lib/stdin.mjs';

// 纠正检测模式
const CORRECTION_PATTERNS = [
  /^(no,?\s+use|don'?t use|actually|that'?s wrong|wrong)/i,
  /^(不对|不要用|实际上|错了|错误)/,
  /^remember[:：]/i,
  /^(perfect|exactly|great approach|yes.*that['']?s right)/i,
  /^(是的.*对的|完全正确|很好的方案)/
];

// 计算置信度
function calculateConfidence(message) {
  let confidence = 0.5;

  // 关键词增强
  if (/remember|记住|以后|下次/.test(message)) confidence += 0.3;
  if (/always|always|必须|一定/.test(message)) confidence += 0.2;
  if (/never|不要|禁止/.test(message)) confidence += 0.2;
  if (/perfect|exactly|完全正确/.test(message)) confidence += 0.15;

  // 上下文减弱
  if (/maybe|可能|或许/.test(message)) confidence -= 0.2;
  if (/sometimes|有时|偶尔/.test(message)) confidence -= 0.1;

  return Math.max(0.1, Math.min(0.99, confidence));
}

// 追加纠正到 questions-log
function appendCorrection(kdevDir, correction) {
  const questionsLogPath = join(kdevDir, 'questions-log.md');

  // 确保 questions-log 存在
  if (!existsSync(questionsLogPath)) {
    const initialContent = `# Questions Log

> 记录所有决策点、澄清问题和纠正历史

## 待解决问题

---

## 已解决问题

---

## 纠正历史

`;
    mkdirSync(kdevDir, { recursive: true });
    appendFileSync(questionsLogPath, initialContent);
  }

  // 追加纠正
  const timestamp = new Date().toISOString().split('T')[0];
  const correctionEntry = `
### C-${Date.now().toString(36)}
- **原问题**: ${correction.original}
- **纠正**: ${correction.corrected}
- **时间**: ${timestamp}
- **置信度**: ${correction.confidence}
`;

  // 追加到 "纠正历史" 部分
  let content = readFileSync(questionsLogPath, 'utf-8');
  if (content.includes('## 纠正历史')) {
    // 在纠正历史部分追加
    const parts = content.split('## 纠正历史');
    const newContent = parts[0] + '## 纠正历史\n' + correctionEntry + parts[1];
    // 需要整体重写，因为 appendFileSync 只能追加末尾
    const fs = require('fs');
    fs.writeFileSync(questionsLogPath, newContent);
  } else {
    // 没有 "纠正历史" 部分，追加到末尾
    appendFileSync(questionsLogPath, '\n## 纠正历史\n' + correctionEntry);
  }
}

// 主函数
async function main() {
  try {
    const input = await readStdin();
    let data = {};
    try { data = JSON.parse(input); } catch {}

    const toolName = data.tool_name || data.toolName || '';
    const toolInput = data.tool_input || data.toolInput || {};
    const toolResult = data.tool_result || data.toolResult || '';
    const workingDirectory = data.cwd || data.directory || process.cwd();

    // 只处理 Edit/Write 工具
    if (!['Edit', 'Write'].includes(toolName)) {
      console.log(JSON.stringify({ continue: true, suppressOutput: true }));
      return;
    }

    // 检测纠正 (从工具输入或结果中)
    const textToCheck = toolInput.new_string || toolInput.content || toolResult || '';
    
    for (const pattern of CORRECTION_PATTERNS) {
      if (pattern.test(textToCheck)) {
        const confidence = calculateConfidence(textToCheck);

        // 只有高置信度才自动记录
        if (confidence >= 0.7) {
          const kdevDir = join(workingDirectory, '.kdev');
          
          // 提取纠正内容 (简化版)
          const correction = {
            original: '用户纠正',
            corrected: textToCheck.slice(0, 200),
            confidence
          };

          appendCorrection(kdevDir, correction);

          console.log(JSON.stringify({
            continue: true,
            hookSpecificOutput: {
              hookEventName: 'PostToolUse',
              additionalContext: `<kdev-correction-detected>
检测到纠正模式 (置信度: ${confidence.toFixed(2)})
已自动记录到 questions-log.md

如果这是错误的记录，运行 /kdev-questions 清理
</kdev-correction-detected>`
            }
          }));
          return;
        }
      }
    }

    console.log(JSON.stringify({ continue: true, suppressOutput: true }));
  } catch (error) {
    console.log(JSON.stringify({ continue: true, suppressOutput: true }));
  }
}

main();

### 5.7 stdin.mjs 辅助模块

```javascript
// scripts/lib/stdin.mjs

/**
 * Timeout-protected stdin reader
 * Prevents hangs on Linux/Windows (参考 OMC issue #240, #524)
 */

export function readStdin(timeoutMs = 5000) {
  return new Promise((resolve) => {
    const chunks = [];
    let settled = false;

    const timeout = setTimeout(() => {
      if (!settled) {
        settled = true;
        process.stdin.removeAllListeners();
        process.stdin.destroy();
        resolve(Buffer.concat(chunks).toString('utf-8'));
      }
    }, timeoutMs);

    process.stdin.on('data', (chunk) => {
      chunks.push(chunk);
    });

    process.stdin.on('end', () => {
      if (!settled) {
        settled = true;
        clearTimeout(timeout);
        resolve(Buffer.concat(chunks).toString('utf-8'));
      }
    });

    process.stdin.on('error', () => {
      if (!settled) {
        settled = true;
        clearTimeout(timeout);
        resolve('');
      }
    });

    // Handle already-ended stream
    if (process.stdin.readableEnded) {
      if (!settled) {
        settled = true;
        clearTimeout(timeout);
        resolve(Buffer.concat(chunks).toString('utf-8'));
      }
    }
  });
}

### 5.10 文件锁机制 (防止并发写入)

```javascript
// lib/file-lock.mjs (简化版)

import { existsSync, unlinkSync, writeFileSync } from 'fs';
import { join } from 'path';

const LOCK_TIMEOUT_MS = 5000;

export async function withFileLock(lockPath, fn, options = {}) {
  const timeout = options.timeoutMs || LOCK_TIMEOUT_MS;
  const startTime = Date.now();

  // 尝试获取锁
  while (Date.now() - startTime < timeout) {
    try {
      // 创建锁文件
      writeFileSync(lockPath, JSON.stringify({
        pid: process.pid,
        timestamp: Date.now()
      }));
      
      // 执行函数
      const result = await fn();
      
      // 释放锁
      try { unlinkSync(lockPath); } catch {}
      
      return result;
    } catch (e) {
      if (e.code === 'EEXIST') {
        // 锁被其他进程持有，等待
        await new Promise(r => setTimeout(r, 100));
        continue;
      }
      throw e;
    }
  }

  // 超时，强制执行 (降级策略)
  return await fn();
}

export function lockPathFor(filePath) {
  return filePath + '.lock';
}

| - | `/kdev-extract-skill` | Skill提取 |

### 6.3 `/kdev-compound` 命令详解

> 对应 Compound Engineering 的 **Compound 步骤**：将学习内容沉淀为可复用知识

**命令设计**:
```bash
/kdev-compound                    # 交互式知识沉淀
/kdev-compound --auto             # 自动从会话历史提取
/kdev-compound --from <file>      # 从指定文件提取
/kdev-compound --refresh          # 刷新过期知识
```

**执行流程**:

### 6.4 `/kdev-compound-refresh` 命令

> 刷新/归档过期的知识，保持记忆系统健康

**命令设计**:
```bash
/kdev-compound-refresh            # 检查所有知识 freshness
/kdev-compound-refresh --archive  # 归档过期知识
/kdev-compound-refresh --prune    # 删除无效知识
```

**Freshness 判断标准**:
| 知识类型 | 新鲜期 | 过期处理 |
|----------|--------|----------|
| gotcha | 90天 | 归档到 archive/ |
| question | 永久 | 仅归档已解决超过30天的 |
| skill | 180天 | 提示用户确认是否仍有效 |
| daily-log | 30天 | 自动清理 |

## 十、总结

### 核心价值 (Compound Engineering 视角)

```
传统工程 → 技术债务积累 → 代码越多越难维护
复利工程 → 知识资产积累 → 代码越多越智能

KDev 记忆层 = 知识资产的"复利账户"
每次 Compound → 存入新知识 → 下次自动复用
```

| 功能 | 价值 | 复利效应 |
|------|------|----------|
| questions-log | 决策追溯 | 避免重复讨论同一问题 |
| gotchas | 踩坑记录 | 新 agent 自动避开已知坑 |
| skills | 模式复用 | 新功能自动复用旧方案 |
| compound 命令 | 知识沉淀 | 每次工作都让下次更容易 |

- **复利效应** - 每次工作让下次更容易 (Compound Engineering)
