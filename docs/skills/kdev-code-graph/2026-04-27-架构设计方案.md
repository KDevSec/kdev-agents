# kdev-code-graph 架构设计方案

## 项目概述

| 属性 | 值 |
|------|-----|
| 插件名称 | kdev-code-graph |
| 版本 | 1.0.0 |
| 创建日期 | 2026-04-27 |
| 目标 | 构建 Claude Code 插件，实现语义级代码图谱，支持需求追溯、变更爆炸半径分析和文档-代码同步检查 |

---

## 1. 问题域分析

### 1.1 核心痛点

| 痛点 | 描述 | 解决方案 |
|------|------|----------|
| 需求追溯困难 | 设计文档（.md/.图片）与代码实现缺乏关联，难以确认需求是否完整实现 | 语义图谱建立 doc → code 追溯链 |
| 变更风险评估 | 修改代码时难以预估影响范围，可能引入隐蔽缺陷 | 爆炸半径分析，可视化依赖链 |
| 文档代码脱节 | 文档更新滞后，代码变更后文档未同步，导致信息不一致 | doc-code-sync Skill 检查同步状态 |

### 1.2 目标用户

- 安全自动化开发团队成员
- 需要进行代码审查的技术人员
- 需求追溯和文档维护的工程师

## 3. 核心模块设计

### 3.1 解析器层 (Parser Layer)

#### 3.1.1 代码解析器 (code_parser.py)

| 属性 | 说明 |
|------|------|
| 解析引擎 | Tree-sitter (支持多语言 AST) |
| 输入 | .py, .js, .ts, .go, .java 等 |
| 输出 | code_node: {type, name, location, calls, depends_on} |
| 节点类型 | function, class, module, import |

**关键数据结构：**

```python
code_node = {
    "id": "uuid",
    "type": "function|class|module",
    "name": "function_name",
    "file_path": "src/module.py",
    "line_start": 42,
    "line_end": 56,
    "calls": ["other_func"],
    "depends_on": ["imported_module"],
    "called_by": [],
    "created_at": "2026-04-27"
}
```

#### 3.1.2 Markdown 解析器 (md_parser.py)

| 属性 | 说明 |
|------|------|
| 解析方式 | Regex + NLP 概念提取 |
| 输入 | .md 文件 (PRD/设计文档/需求文档) |
| 输出 | doc_node: {type, concepts, sections, references} |
| 概念提取 | 标题 → 概念名，内容 → 描述，引用 → 关联 |

**关键数据结构：**

```python
doc_node = {
    "id": "uuid",
    "type": "prd|design|requirement|spec",
    "name": "auth_design.md",
    "concepts": ["AuthService", "TokenValidation", "Session"],
    "sections": [
        {"title": "认证流程", "level": 2, "content": "..."}
    ],
    "references": ["api_spec.md", "security_rules.md"],
    "created_at": "2026-04-27"
}
```

#### 3.1.3 图片解析器 (image_parser.py)

| 属性 | 说明 |
|------|------|
| 解析方式 | Claude Vision API (主) / OCR (备) |
| 输入 | .png, .jpg, .webp |
| 输出 | image_node: {type, concepts, components, relationships} |
| 图片类型识别 | diagram, flowchart, screenshot, wireframe |

**关键数据结构：**

```python
image_node = {
    "id": "uuid",
    "type": "diagram|flowchart|screenshot|wireframe",
    "name": "auth_flow.png",
    "file_path": "docs/images/auth_flow.png",
    "concepts": ["LoginFlow", "AuthService", "TokenStore"],
    "components": [
        {"name": "UserService", "type": "service"}
    ],
    "relationships": [
        {"from": "UserService", "to": "AuthService", "type": "calls"}
    ],
    "created_at": "2026-04-27"
}
```

### 3.2 语义提取层 (Semantic Layer)

#### 3.2.1 LLM 语义关联器 (extractor.py)

| 属性 | 说明 |
|------|------|
| 模型 | Claude Sonnet 4.6 |
| 任务类型 | 语义匹配 + 信心评分 |
| 输入 | doc_node / image_node + code_nodes |
| 输出 | semantic_edge: {source, target, confidence, relation_type} |

**信心评分算法：**

```python
confidence_score = {
    "exact_match": 1.0,       # 名称完全匹配
    "semantic_match": 0.7-0.9, # LLM 判断语义相关
    "context_match": 0.5-0.7,  # 通过上下文推断
    "fuzzy_match": 0.3-0.5    # 模糊匹配
}
```

### 3.3 图存储层 (Graph Layer)

#### 3.3.1 SQLite 图存储 (store.py)

**表结构设计：**

```sql
-- 节点表
CREATE TABLE nodes (
    id TEXT PRIMARY KEY,
    type TEXT NOT NULL,           -- 'code'|'doc'|'image'
    subtype TEXT,                 -- 'function'|'class'|'prd'...
    name TEXT NOT NULL,
    file_path TEXT,
    metadata JSON,                -- 扩展属性
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- 边表
CREATE TABLE edges (
    id TEXT PRIMARY KEY,
    source_id TEXT NOT NULL,
    target_id TEXT NOT NULL,
    relation_type TEXT NOT NULL,  -- 'calls'|'depends'|'semantic'
    confidence REAL DEFAULT 1.0,
    metadata JSON,
    created_at TIMESTAMP,
    FOREIGN KEY (source_id) REFERENCES nodes(id),
    FOREIGN KEY (target_id) REFERENCES nodes(id)
);

-- 变更记录表
CREATE TABLE changes (
    id TEXT PRIMARY KEY,
    node_id TEXT NOT NULL,
    change_type TEXT NOT NULL,    -- 'created'|'modified'|'deleted'
    file_path TEXT NOT NULL,
    timestamp TIMESTAMP,
    metadata JSON
);
```

#### 3.3.2 爆炸半径分析 (blast_radius.py)

| 属性 | 说明 |
|------|------|
| 算法 | BFS (广度优先搜索) |
| 输入 | 变更节点 ID + 深度限制 |
| 输出 | impact_set: {direct, indirect, confidence_order} |

**分析流程：**

```
变更节点 (A)
    │
    ▼
┌─────────────────────────────────────┐
│  Level 1: 直接调用者 (B, C, D)       │
│  - edge_type: calls                 │
│  - confidence: 1.0                  │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  Level 2: 间接依赖 (E, F, G)         │
│  - edge_type: depends|semantic      │
│  - confidence: 0.7-0.9              │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  Level 3: 语义关联 (H, I)            │
│  - edge_type: semantic              │
│  - confidence: 0.5-0.7              │
└─────────────────────────────────────┘
```

## 4. MCP Tools 设计

### 4.1 Tool 清单

| Tool 名称 | 功能 | 输入 | 输出 |
|-----------|------|------|------|
| `build_graph` | 构建/重建语义图谱 | project_root | {nodes_count, edges_count, status} |
| `semantic_query` | 语义查询 | concept, limit | {nodes, edges, confidence} |
| `doc_code_trace` | 文档到代码追溯 | doc_name | {code_nodes, implementation_status} |
| `detect_changes` | 变更检测 | file_path(s) | {changed_nodes, new_edges} |
| `get_impact_radius` | 爆炸半径分析 | node_id, depth | {impact_set, visualization} |

### 4.2 Tool 详细设计

#### semantic_query Tool

```python
def semantic_query(concept: str, limit: int = 10) -> dict:
    """
    根据概念查询相关代码节点。

    Args:
        concept: 搜索概念 (如 "AuthService", "认证流程")
        limit: 返回结果数量限制

    Returns:
        {
            "nodes": [
                {
                    "id": "...",
                    "type": "code",
                    "name": "AuthService",
                    "file_path": "src/auth/service.py",
                    "confidence": 0.95
                }
            ],
            "edges": [...],
            "query_time_ms": 42
        }
    """
```

#### get_impact_radius Tool

```python
def get_impact_radius(node_id: str, depth: int = 3) -> dict:
    """
    计算变更爆炸半径。

    Args:
        node_id: 变更节点 ID
        depth: 搜索深度 (默认3层)

    Returns:
        {
            "impact_set": {
                "level_1": [...],  # 直接影响
                "level_2": [...],  # 间接影响
                "level_3": [...]   # 语义影响
            },
            "total_nodes": 15,
            "high_confidence_count": 8,
            "visualization": "ASCII tree or Mermaid"
        }
    """
```

```
    │
    ▼

    ▼
Step 3: doc_code_trace("auth_design.md")
    │
    ▼
Step 4: 输出追溯报告
    - 需求 → 代码映射
    - 实现状态 (已实现/部分实现/未实现)

```

|----------|------|------|

| 触发条件 | 动作 | 目的 |

## 7. 支持格式矩阵

| 格式 | 解析方式 | 提取内容 | 关联方式 | 信心评分 |
|------|----------|----------|----------|----------|
| .py/.js/.go | Tree-sitter AST | 函数、类、调用关系 | 直接关联 | 1.0 |
| .md | Regex + NLP | 概念、需求、设计 | LLM语义 | 0.7-0.9 |
| .png/.jpg | Vision API / OCR | 设计组件、流程步骤 | LLM语义 | 0.5-0.8 |

## 8. 技术选型

| 组件 | 技术选择 | 选择理由 |
|------|----------|----------|
| AST解析 | Tree-sitter | 多语言支持、增量解析、错误容忍 |
| 图存储 | SQLite | 轻量级、无需外部服务、查询性能好 |
| LLM API | Claude Sonnet 4.6 | 高质量语义理解、支持 Vision |
| Vision解析 | Claude Vision | 直接集成，无需额外 OCR 库 |
| MCP协议 | stdio | 简单可靠，适合本地工具 |

## 12. 安全与性能考虑

### 12.1 安全设计

| 威胁 | 防护措施 |
|------|----------|
| API Key 泄露 | 环境变量存储，不写入配置文件 |
| 代码泄露 | 本地处理，仅向 Claude API 发送必要片段 |
| 图谱篡改 | SQLite 文件权限控制 |

### 12.2 性能优化

| 策略 | 说明 |
|------|------|
| 增量解析 | Tree-sitter 支持增量 AST 解析 |
| 缓存机制 | LLM 结果缓存，避免重复调用 |
| 批量处理 | 多文件并发解析 |
| 深度限制 | 爆炸半径分析限制深度，防止无限扩散 |

## 附录 C: 参考 code-review-graph 的架构模式

基于对 [code-review-graph](https://github.com/tirth8205/code-review-graph) 项目的分析，本设计可直接复用以下架构模式。

### C.1 可直接复用的核心模块

| 模块 | 文件 | 复用方式 |
