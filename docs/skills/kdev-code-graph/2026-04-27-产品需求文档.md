## 1. 产品概述

### 1.1 产品定位

**KDevSec Code Graph** 是一个 Claude Code 插件，为开发者提供智能化的代码-文档语义关联和变更影响分析能力。它结合知识图谱技术、语义分析和爆炸半径分析，帮助开发者在编码、修 Bug、需求追溯等场景中高效决策。

### 1.2 核心价值

| 价值点 | 说明 |
|--------|------|
| **需求追溯** | 自动建立 spec/design.md → code 的语义关联，验证需求是否已实现 |
| **变更影响分析** | 爆炸半径分析，精确评估代码变更的影响范围 |
| **文档-代码一致性检查** | 检测设计文档与实际代码实现的偏差 |
| **多模态支持** | 支持 Markdown 文档 + 设计图片的语义解析 |
| **高效 Token 使用** | 图谱查询比全文件扫描减少 6.8×~71.5× token |

### 1.3 目标用户

| 用户类型 | 核心需求 |
|----------|----------|
| **后端开发者** | 编码、修 Bug、理解调用链 |
| **前端开发者** | 理解 UI 设计图与代码的对应关系 |
| **技术 Leader** | Code Review、架构决策、风险评估 |
| **新接手项目的开发者** | 快速理解代码库结构和设计意图 |

## 3. 用户场景

### 3.1 场景 1：需求实现验证

**用户故事**: 作为开发者，我需要确认 spec.md 中描述的"用户登录功能"是否已在代码中实现。

**流程**:
```
1. 用户输入: /KDevSec-trace 检查用户登录需求是否已实现
2. 插件调用: KDevSec_query("用户登录")
3. 返回结果:
   - spec.md#用户登录 [EXTRACTED, confidence=1.0]
     └─ UserService.login() [INFERRED, confidence=0.85]
     └─ AuthController.authenticate() [INFERRED, confidence=0.72]
   - 实现状态: PARTIAL（登录逻辑已实现，但缺少验证码功能）
4. 建议: 需要在 UserService 中添加验证码验证逻辑
```

### 3.2 场景 2：Bug 修复影响分析

**用户故事**: 作为开发者，我需要修复 UserService.login() 的一个 bug，想知道会影响哪些其他代码。

**流程**:
```
1. 用户输入: /code-review-enhanced 分析 UserService.login 的变更影响
2. 插件调用: detect_changes(file="UserService.py")
3. 返回结果:
   - 变更节点: UserService.login()
   - 爆炸半径:
     └─ AuthController (调用者)
     └─ SessionManager (依赖者)
     └─ test_login.py (测试)
     └─ test_auth_flow.py (测试)
   - 风险评分: MEDIUM (有测试覆盖，但 SessionManager 无测试)
4. 建议: 修改后需补充 SessionManager 的单元测试
```

### 3.3 场景 3：设计图与代码对照

**用户故事**: 作为前端开发者，我有一张 UI 设计图，想知道哪些组件已经实现，哪些还没实现。

**流程**:
```
1. 用户输入: /doc-code-sync 检查设计图 login-ui.png 的实现情况
2. 插件调用: doc_code_trace(doc="login-ui.png")
3. 图片解析:
   - 提取组件: LoginForm, SubmitButton, CaptchaInput, ErrorTip
4. 返回结果:
   - LoginForm → components/LoginForm.tsx [confidence=0.9] ✅
   - SubmitButton → components/Button.tsx [confidence=0.8] ✅
   - CaptchaInput → ❌ 未找到对应实现
   - ErrorTip → components/Toast.tsx [confidence=0.6] ⚠️ 部分
5. 建议: 需要实现 CaptchaInput 组件
```

## 4. 数据模型

### 4.1 节点类型

| 节点类型 | 属性 | 来源 |
|----------|------|------|
| `function` | name, file, line, signature, docstring | Tree-sitter |
| `class` | name, file, line, methods, inherits | Tree-sitter |
| `module` | name, file, exports | Tree-sitter |
| `test_file` | name, file, tests | Tree-sitter 检测 |
| `document` | name, file, title, concepts | MD Parser |
| `image` | name, file, type, components | Vision API |

### 4.2 边类型

| 边类型 | 源节点 | 目标节点 | confidence | 来源 |
|----------|--------|----------|------------|------|
| `calls` | function | function | 1.0 (EXTRACTED) | AST |
| `imports` | module | module/function | 1.0 (EXTRACTED) | AST |
| `inherits` | class | class | 1.0 (EXTRACTED) | AST |
| `tests` | test_file | function | 1.0 (EXTRACTED) | AST 检测 |
| `KDevSecally_similar` | document/image | function/class | 0.0~1.0 (INFERRED) | LLM |
| `implements` | function | document.requirement | 0.0~1.0 (INFERRED) | LLM |
| `contains` | document | paragraph/concept | 1.0 (EXTRACTED) | MD Parser |

### 4.3 数据存储结构

```sql
-- SQLite 表结构
CREATE TABLE nodes (
    id INTEGER PRIMARY KEY,
    type TEXT NOT NULL,          -- function/class/module/document/image
    name TEXT NOT NULL,
    file TEXT,
    line INTEGER,
    properties JSON,             -- 扩展属性
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

CREATE TABLE edges (
    id INTEGER PRIMARY KEY,
    source_id INTEGER NOT NULL,
    target_id INTEGER NOT NULL,
    type TEXT NOT NULL,          -- calls/imports/inherits/tests/KDevSecally_similar
    confidence REAL DEFAULT 1.0, -- 0.0~1.0
    source_type TEXT,            -- EXTRACTED/INFERRED/AMBIGUOUS
    properties JSON,
    created_at TIMESTAMP
);

CREATE INDEX idx_nodes_type ON nodes(type);
CREATE INDEX idx_nodes_name ON nodes(name);
CREATE INDEX idx_edges_source ON edges(source_id);
CREATE INDEX idx_edges_target ON edges(target_id);
CREATE INDEX idx_edges_type ON edges(type);
```

## 5. 非功能需求

### 5.1 性能要求

| 指标 | 要求 |
|------|------|
| 初始构建时间 | 500文件项目 <10秒 |
| 增量更新时间 | <2秒 |
| 单次查询响应 | <500ms |
| 内存占用 | <200MB (500文件项目) |
| 图谱存储大小 | <50MB (500文件项目) |

### 5.2 安全要求

| 要求 | 说明 |
|------|------|
| 本地存储 | 所有数据存储在本地 SQLite，不上传云端 |
| API 密钥管理 | LLM API 密钥通过环境变量注入，不硬编码 |
| 图片处理 | Vision API 调用时图片不缓存到云端 |
| 代码内容 | Tree-sitter 解析的代码内容不上传 |

### 5.3 可用性要求

| 要求 | 说明 |
|------|------|
| 安装方式 | 支持 `/plugins install` 从 Git/npm 安装 |
| 配置自动化 | 安装后自动配置 MCP Server + Hooks |
| 错误恢复 | 增量更新失败时回滚到上一个稳定状态 |
| 兼容性 | Windows/Mac/Linux 全平台支持 |

## 6. 约束条件

### 6.1 技术约束

| 约束 | 说明 |

| Python 版本 | >=3.10 |

| 约束 | 说明 |

## 9. 风险分析

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| LLM API 成本过高 | 用户流失 | 中 | 提供本地 embedding 作为可选替代 |
| Vision API 图片处理慢 | 体验下降 | 中 | 图片处理异步化，显示进度 |
| Tree-sitter 语言支持不完整 | 功能受限 | 低 | 优先支持主流语言，其他语言可禁用 |
| Claude Code 插件 API 变化 | 兼容性问题 | 低 | 关注官方更新，快速适配 |
| SQLite 性能瓶颈 | 大项目慢 | 低 | 提供图谱分片/压缩选项 |
