## Phase 2: 文档解析 + 语义关联

### 2.1 Markdown 解析器

参考：`references/graphify/graphify/extract.py` (多模态提取模式)

```python
# mcp-server/parser/md_parser.py
import re
from pathlib import Path

def extract_md_concepts(file_path: Path) -> dict:
    """提取 Markdown 文档中的概念"""
    content = file_path.read_text()

    # 提取标题作为概念
    headings = re.findall(r'^#+ (.+)$', content, re.MULTILINE)

    # 提取 #WHY:, #NOTE: 等设计决策
    decisions = re.findall(r'#(WHY|NOTE|IMPORTANT): (.+)', content)

    return {
        "concepts": headings,
        "decisions": decisions,
        "doc_type": classify_doc_type(file_path)  # prd/design/spec
    }
```

### 2.2 LLM 语义关联器

```python
# mcp-server/KDevSec/extractor.py
import anthropic

def extract_KDevSec_links(doc_concepts: list, code_nodes: list) -> list:
    """使用 LLM 提取文档概念与代码的语义关联"""

    prompt = f"""
    文档概念: {doc_concepts}
    代码节点: {[n.name for n in code_nodes]}

    判断每个概念与哪个代码节点相关，返回 JSON:
    - concept: 概念名
    - code_node: 匹配的代码节点 qualified_name
    - confidence: 0.0-1.0 信心评分
    - reason: 关联原因（一句话）
    """

    response = anthropic_client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1024,
        messages=[{"role": "user", "content": prompt}]
    )

    return parse_KDevSec_response(response)
```
