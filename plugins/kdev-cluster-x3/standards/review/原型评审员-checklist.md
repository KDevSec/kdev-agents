# 原型评审员 checklist（R2 阻断）

> 评审对象：`.kdev/handoffs/reqs/prototype/`
> 评审结论：PASS / FAIL（FAIL 必须列具体问题）

## R2 必检项（10 条 / 1 项 FAIL = 总体 FAIL）

1. [ ] **UI 完整性**：prototype/ 目录下覆盖所有 AR 涉及的页面 / 组件，无缺失视图。
2. [ ] **项目 UED 规范遵守**：色彩、字体、间距、图标风格与项目 UED 规范一致，无擅自引入新样式体系。
3. [ ] **关键流转正确**：主干用户流（注册/登录/核心业务）的页面跳转路径可完整走通，无断链或死页面。
4. [ ] **边界态覆盖**：每个关键列表 / 表格均有空态（empty state）原型；分页 / 滚动边界有设计说明。
5. [ ] **错误态覆盖**：表单验证错误、网络失败、权限不足等异常场景均有独立原型帧或标注。
6. [ ] **响应式适配**：若产品目标含多分辨率，至少提供两种断点下的布局原型；否则需注明「仅桌面端」。
7. [ ] **可访问性标注**：关键交互元素标注 aria-label / 焦点顺序，色对比度满足 WCAG AA 标准（对比度 ≥ 4.5:1）。
8. [ ] **与 AR 对应**：prototype/ 内每个原型帧或页面标注对应的 AR 编号，100% AR 可反查到原型。
9. [ ] **交互说明完整**：弹窗、抽屉、下拉、Tooltip 等非静态交互均有文字说明或 Overlay 标注。
10. [ ] **无 TBD / TODO / FIXME** 占位标注。
