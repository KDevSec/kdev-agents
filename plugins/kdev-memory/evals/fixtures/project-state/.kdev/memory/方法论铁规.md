# 方法论铁规

## commit 粒度
triggers: ["git commit", "commit 消息", "提交代码"]

一事一 commit，不要把 PRD 扩展 + 基线计划 + brainstorm 三件事打包了。
用户多次反馈过"这个 commit 混进了好几件事"，立为铁规：每次 commit 前
主动问一下"这次改动可以拆成几个逻辑 commit 吗"。

## API 设计
triggers: ["设计 API", "新 endpoint", "REST endpoint"]

所有新 endpoint 必须：
- 带版本前缀 /v1/、/v2/
- 返回值包 `{data, error, meta}` 三段式
- 错误码用 HTTP code + 自定义 `error.code` 字符串双轨
