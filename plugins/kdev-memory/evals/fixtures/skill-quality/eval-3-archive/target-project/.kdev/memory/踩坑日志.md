# 踩坑日志

## G-001: nvm 切换 Node 版本后 pnpm 丢失全局包
triggers: ["nvm", "pnpm", "全局包"]
日期：2026-02-06
现象：切到 Node 20 后 `pnpm` 命令找不到
原因：pnpm 装在 Node 18 的全局目录
解决：每个 Node 版本单独 `npm i -g pnpm` 或用 corepack

## G-002: Next.js app router 的 cookie 设置必须在 Server Action
triggers: ["Next.js cookie", "server action"]
日期：2026-02-19
现象：Client Component 里写 cookie 无效
原因：RSC 架构限制，cookie 只能在 Server Action / Route Handler
解决：把 OAuth callback 改到 Route Handler

## G-003: Prisma migrate reset 在 prod 是灾难
triggers: ["prisma", "migrate reset"]
日期：2026-03-03
现象：差点在 prod 跑 reset
原因：脚本 alias 没区分环境
解决：加 NODE_ENV guard + prod 走 migrate deploy

## G-004: pg_trgm 索引只加速 % 操作符，不加速 LIKE
triggers: ["pg_trgm", "LIKE", "搜索优化"]
日期：2026-04-08
现象：加了索引但 LIKE 查询没变快
原因：需要用 % 模糊匹配操作符，不是 LIKE
解决：查询改成 `WHERE col % 'keyword'`

## G-005: scws 扩展在 Alpine 容器装不上
triggers: ["scws", "Alpine", "postgres extension"]
日期：2026-04-15
现象：docker 镜像装 scws 报 compiler not found
原因：Alpine 的 musl 和 scws 源码不兼容
解决：切到 Debian slim 镜像

## G-006: ts_rank 在没 GIN 索引的列上很慢
triggers: ["ts_rank", "GIN index"]
日期：2026-04-21
现象：排序查询 P99 500ms
原因：ts_rank 对没 GIN 索引的 tsvector 列做顺序扫描
解决：给 tsvector 列加 GIN 索引后降到 30ms
