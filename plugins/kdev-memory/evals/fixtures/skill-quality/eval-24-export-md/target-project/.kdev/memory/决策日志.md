# 决策日志

## Q-001

日期：2026-05-10
问题：Step 编号用全局递增还是迭代内递增？
选项：A. 全局递增（1, 2, 3…不跨迭代重置）；B. 迭代内递增（每次 Sprint 重新从 1 开始）
用户选择：A. 全局递增
理由：全局唯一编号便于跨文件引用和日志检索，避免 "Sprint 2 Step 3" 和 "Sprint 3 Step 3" 产生混淆。

## Q-002

日期：2026-05-11
问题：踩坑日志是否自动触发切档（超过 50 条时）？
选项：A. 自动切档（hook 检测超出阈值后直接移文件）；B. 人工触发（用户手动说"切档"才搬）
用户选择：B. 人工触发
理由：自动切档有误操作风险，宁可让用户主动说"切档"，hook 只负责提醒。
