## R-001: 字段对齐脱钩（PRD vs 代码）
日期：2026-04-10
项目：demo-api
触发：执行日志 Step 6 评分差值 -2（模型 5/5，用户 3/5）
模型自评原话："代码一次跑通，端到端 OK"
用户评价原话："接口返回的字段名跟 PRD 对不上，又得我一个个看回去"
事实段：工具调用约 18，0 报错，0 绕路
初步诊断：模型把"能跑通"当完成，用户要的是"对齐 PRD 字段名"——两套验收标准
可能的改进方向：
  - 写代码前先把 PRD 字段名列出来
  - 或做个 schema-diff skill
